1. Open files identified

Command:
lsof

Output: All the open files in screenshot. Too large to type here.

Explanation:
The lsof command lists all files currently opened by running processes in the Linux system. It helps identify which files are in use and which process is accessing them.

2. File descriptor observations

Command:
ls -l /proc/$$/fd

Output:
total 0
lrwx------ 1 shrey shrey 64 Jul  3 15:52 0 -> /dev/pts/0
lrwx------ 1 shrey shrey 64 Jul  3 15:52 1 -> /dev/pts/0
lrwx------ 1 shrey shrey 64 Jul  3 15:52 2 -> /dev/pts/0
lrwx------ 1 shrey shrey 64 Jul  3 15:52 255 -> /dev/pts/0

Explanation:
The command displays the file descriptors associated with the current shell process. It shows how Linux manages standard input, standard output, standard error, and other open files.

3. Output redirection and error redirection results

Command:
echo "Hello Linux" > output.txt
cat output.txt
ls nonexistentfile 2> error.txt
cat error.txt

Output:
Hello
ls: cannot access 'nonexistentfile': No such file or directory

Explanation:
The `>` operator redirects standard output to a file, while `2>` redirects standard error to a separate file. This allows normal output and error messages to be handled independently.

5. Explanation of how Linux manages file I/O

Explanation:
Linux manages file input and output through file descriptors, where standard input, standard output, and standard error are assigned descriptor numbers 0, 1, and 2 respectively. It also supports redirection and resource limits to efficiently control how processes access files and system resources.
